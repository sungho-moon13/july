# 게임 개발 계획서

## 1. 게임 주제

- 게임 제목: 디지털 창고 관리자
- 핵심 주제: 컴퓨터의 저장 용량을 쉽게 이해하는 게임
- 한 줄 소개: 컴퓨터의 저장 용량을 쉽게 이해할 수 있다.
- 이 게임을 만들고 싶은 이유: 컴퓨터의 저장 용량 개념이 헷갈리기 때문에 만들고 싶다.

## 2. 학습 목표

- 플레이어가 게임을 통해 알게 될 내용:
- 꼭 설명하고 싶은 저장 용량 개념:
  - bit: 비트. 컴퓨터가 정보를 저장하는 가장 작은 단위로, 0 또는 1 하나를 나타내는 단위.
  - byte: 바이트. 8개의 bit가 모인 단위로, 글자나 작은 정보를 저장할 때 기본으로 쓰는 단위.
  - KB: 킬로바이트. 작은 문서 파일이나 아주 작은 이미지의 크기를 나타낼 때 쓰는 단위.
  - MB: 메가바이트. 사진, 음악 파일, 짧은 영상, 앱의 크기를 나타낼 때 자주 쓰는 단위.
  - GB: 기가바이트. 게임, 영화, 스마트폰 저장 공간, USB 저장 용량을 나타낼 때 많이 쓰는 단위.
  - TB: 테라바이트. 컴퓨터 하드디스크, 외장하드처럼 아주 큰 저장 공간을 나타낼 때 쓰는 단위.
- 저장 용량을 비유할 대상: 물건을 보관하는 디지털 창고
- 게임을 끝낸 뒤 플레이어가 할 수 있어야 하는 것: 컴퓨터의 저장 용량의 개념을 이해 하는것.

## 3. 대상 플레이어

- 주요 대상 연령: 초등학교 고학년 ~ 중학교 1학년
- 컴퓨터 지식 수준: 컴츄터 지식이 없어도 가능
- 플레이 시간이 얼마나 걸리면 좋을지: 한 스테이지 당 5~6분
- 플레이어가 어려워할 수 있는 부분: 알맞은 창고에 파일 아이템을 넣는 것

## 4. 게임 장르와 방식

- 게임 장르: 디지털 창고 정리 게임
- 조작 방식: 마우스로 파일 아이템을 끌어서 알맞은 저장 공간에 넣기
- 화면 구성: 왼쪽에는 정리해야 할 파일, 오른쪽에는 KB, MB, GB, TB 창고를 배치
- 플레이 흐름: 파일의 종류와 크기를 확인한 뒤 알맞은 용량 창고에 분류하기

## 5. 게임 세계관과 이야기

- 배경 설정: 컴퓨터 안에 있는 디지털 창고가 파일들로 어지럽혀진 상황
- 주인공: 디지털 창고를 정리하는 초보 저장소 관리자
- 플레이어의 역할: 여러 파일의 크기를 보고 알맞은 용량 창고에 정리하는 역할
- 저장 용량을 설명하기 위한 이야기 설정: 파일마다 크기가 다르기 때문에 작은 파일은 작은 창고에, 큰 파일은 큰 창고에 넣어야 함
- 게임 시작 상황: 사진, 문서, 음악, 영상, 게임 파일이 뒤섞여 있어서 창고가 복잡해짐
- 게임 엔딩 상황: 파일들이 알맞은 창고에 정리되고, 플레이어가 KB, MB, GB, TB의 차이를 이해하게 됨

## 6. 핵심 게임 규칙

- 플레이어가 해야 하는 일: 사진, 문서, 음악, 영상, 게임 파일을 알맞은 크기의 디지털 창고에 정리하기
- 저장 용량 개념이 규칙에 들어가는 방법: 작은 파일은 KB 또는 MB 창고에, 큰 파일은 GB 또는 TB 창고에 넣어야 점수를 얻음
- 점수 계산 방식: 파일마다 5점씩
- 제한 시간 또는 제한 횟수: 5분 제한시간
- 난이도가 올라가는 방식: 아이템 파일이 점점 많아진다.
